  • 문제 대규모 트랜잭션 환경에서 DB Query 부하 급증이 예상되었습니다.
  • 해결 불필요한 Join 제거를 위한 테이블 구조 개선과 서버 메모리 캐싱 전략을 적용했습니다.
  • 성과 일 60만 건 수준의 DB Query 부하를 약 70~75% 절감했습니다.

데이터 접근성

MCP + Claude 기반 SQL 변환 인터페이스

  • 문제 비개발 인력의 DB 조회 수요는 높았지만 SQL 작성 난도로 인해 데이터 확인과 고객 대응이 지연됐습니다.
  • 해결 MCP 기반 스키마 추상화, 자연어-SQL 변환, 허용 Query 패턴 제한, SQL 검증 로직을 구현했습니다.
  • 성과 유지보수 부서 2명이 자주 요청하던 쿼리를 사내 출결/성과 등록 사이트에서 직접 조회할 수 있게 연동해, 개발자 쿼리 요청 문의를 사실상 0건으로 줄였습니다.

  • 문제 기능별 구현 방식이 혼재되어 화면-백엔드-EAI 연동 일관성과 확장성이 떨어졌습니다.

프로젝트 경험

Whisper 기반 STT를 ARS 시스템에 적용한 사내 프로젝트

음성 데이터를 텍스트로 변환해 상담 품질 분석과 키워드 기반 활용을 목표로 진행한 프로젝트입니다. 초기에는 실시간 음성 처리 구조와 데이터 활용 체계가 부족해 운영 가능한 형태로 연결하기 어려운 상황이었습니다.

이를 해결하기 위해 Media Server의 Rx/Tx 음성 스트림을 분리 검출하고 STT 엔진으로 전달하는 실시간 파이프라인을 설계·구현했습니다. 또한 약 1,000건의 녹취 파일을 대상으로 STT 처리 자동화를 구축하고, 키워드 검출 결과를 정렬·분석해 실제 활용 가능한 키워드를 선별했습니다.

결과적으로 상용화까지 이어지지는 않았지만, 실시간 음성 처리 아키텍처 설계와 데이터 기반 기능 개선 전반을 경험하며 음성·데이터 파이프라인에 대한 실무 역량을 강화했습니다.

관심 기술과 공부 방향

최근에는 Claude 기반의 Claude Code와 Cursor를 활용한 개발 방식에 관심을 두고, 개발 생산성을 높이는 실전 적용 방법을 지속적으로 탐색하고 있습니다. 단순 코드 보조를 넘어 이른바 바이브 코딩 방식으로 기능을 빠르게 구현하고 검증하는 흐름을 업무에 적용하고 있습니다.

또한 AI Agent 구조와 Model Context Protocol 기반 연동 방식에 관심을 가지고 Python 학습과 함께 관련 기능을 직접 구현했습니다. Cursor를 활용해 Gmail과 연동되는 AI Agent를 개발하고, 고객사 회의 일정 및 요구사항을 자동 정리해 Google 스프레드시트에 기록하는 기능을 구현했습니다.

이를 통해 단순 기능 개발을 넘어 반복 업무를 자동화하고 데이터 흐름을 개선하는 방향의 가능성을 확인했으며, 앞으로도 AI 기반 시스템 설계 및 활용 역량을 지속적으로 강화해 나가고자 합니다.
